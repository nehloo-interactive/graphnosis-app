Graphnosis can't find Ollama yet. Ollama is a free, open-source app that runs the AI model on your own computer. It takes about three steps:

  1. Install Ollama. Download it from ollama.com → Open the downloaded file and follow the installer — on a Mac, drag Ollama into your Applications folder; on Windows, run the installer.
  2. Start Ollama. Open the Ollama app once. It then runs quietly in the background — look for its icon in the menu bar (Mac) or system tray (Windows). There's no window to keep open; that's normal. Ollama just needs to be running for Graphnosis to use it.
  3. Connect. Come back here and click Recheck above. When the status turns green, you can download a model — Graphnosis walks you through it.

Ollama is free and open source, and runs entirely on your machine — your memories never leave your device. Graphnosis only downloads the model through your own copy of Ollama; it never installs anything for you.

Prefer the Terminal?

Once Ollama is installed you can download and run a model from a terminal — Graphnosis detects it automatically. Open Terminal (macOS) or PowerShell (Windows) and run: 

# 1. Download the recommended model (one-time, ~2 GB)
ollama pull llama3.2:3b

# 2. Run it — this also starts Ollama's background service
ollama run llama3.2:3b

# Lighter option for older machines (~800 MB)
ollama pull llama3.2:1b

# See which models you have / confirm Ollama is up
ollama list

Ollama serves on http://127.0.0.1:11434. After a model is pulled and Ollama is running, click Recheck above — the status turns green and the model appears in the Active model list.

Memory storage

Cached source content lets the App rebuild your graph from the encrypted op-log after accidental data loss — including for pasted text and AI-conversation snippets that aren't stored anywhere else. Cache files are encrypted with the same passphrase as your graph.

When you Forget a source

Forgotten memories are soft-deleted first — confidence drops to zero and they never appear in recall. They stay on disk until physically purged, which gives you a chance to undo and lets the graph stay fast.

Ingest

How aggressively the SDK splits documents into memory nodes, and how many texts it groups into each embedding model call. Sensible defaults cover most users; tune if ingest feels slow or recall feels imprecise. Changes apply to the next file you ingest — existing nodes keep their previous shape.

File-source auto-reingest

When a file you've ingested changes on disk, Graphnosis can re-read it automatically — same flow as the Reingest button in the Sources tab. Useful for Obsidian notes, daily journals, or any file you actively edit. Off by default: enabling means every save fires a re-chunk + re-embed pass, which is fine for small notes but heavier for big PDFs.

Clipboard capture

When enabled, Graphnosis watches your clipboard while it's in focus. Any text you copy that's longer than a sentence is offered for saving as a memory. Disabled by default — you control what goes into your graph.

Engrams in this cortex

Archive an engram to hide it from the picker without touching its data. Delete permanently removes all memories, sources, and embeddings — this cannot be undone.

Each engram carries a sensitivity tier — set it with the dropdown on each row below. The tier controls how much your connected AI clients can pull from that engram:

  • Public — full access. The AI can pull up to 4 000 tokens per recall and may surface these memories proactively, without being asked. Best for reference material, documentation, and notes you don't mind the AI volunteering.
  • Personal — access only when you explicitly ask. Up to 2 000 tokens per recall; no proactive injection. Best for journal entries, personal notes, and work summaries. This is the default for new engrams.
  • Sensitive — the AI is fully blocked. It receives 0 results from this engram for any query and is never even told it exists. You can still browse and search it yourself in Graphnosis. Best for health records, financial data, or anything strictly private.

Local & offline sources

Anything you can pipe into a file or a webhook can feed Graphnosis — no cloud round-trip, no API keys.

The connectors under Data sources all talk to cloud SaaS. But your cortex doesn't have to. Graphnosis runs entirely on your machine — and so do these patterns for plugging in off-the-grid data. Use them as starting points; the full guide has scripts and step-by-step setup for each.

📁 Local files & folders
Drag files onto the app, or use the Obsidian / AI Context Files / GBrain connectors below. Folders re-scan on schedule.
🏠 Smart-home (Home Assistant, MQTT)
Point the Webhook connector at a small bridge script that subscribes to your MQTT broker and POSTs events. Or just watch the broker's log file as plain text.
🌡 Sensors / IoT / lab / agricultural
Anything emitting JSON lines to a file or hitting a webhook becomes a source. A 5-line script reading a serial sensor and POSTing once a minute is enough.
💾 NAS / network drives
Mount the share as a local folder; treat it as Local files. Graphnosis re-scans on schedule, same as Obsidian.
📄 Scanned PDFs / paper records
Drop the PDF onto the app — OCR runs locally, no cloud round-trip. Handwritten notes, intake forms, contracts, grant reports — all become searchable.
🗄 Local databases (SQLite, Postgres on LAN)
Export to JSON or CSV + drop, or point the Webhook connector at a small query-and-POST script on a cron.
📝 Local notes apps (Apple Notes, Bear, Logseq, Notion local cache)
Most have CLI export tools — pipe the output to a watched folder and Graphnosis ingests new entries automatically.
📻 Router syslog, DVR transcripts, audio recordings
Any structured log on disk + a watcher script. Audio: transcribe locally with whisper.cpp first; drop the transcript folder.

Everything in this list runs end-to-end on your machine. No cloud round-trip, no API keys, no data leaving the network. The only thing Graphnosis needs is a file or an HTTP webhook — what's on the other side of either is up to you.
