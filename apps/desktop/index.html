Settings

Per-Cortex preferences. Stored at <cortex>/settings.json.

Content cache

Cached source content lets the App rebuild your graph from the encrypted op-log after accidental data loss — including for pasted text and AI-conversation snippets that aren't stored anywhere else. Cache files are encrypted with the same passphrase as your graph.

MCP relay timings

How long the small relay process (the bridge between Claude Desktop and your Cortex) waits before giving up. Changes apply the next time a client spawns or reconnects a relay — current Claude sessions keep their old values.

seconds (2–120, default 10)
seconds (5–86400, default 86400 = 24h)

When you Forget a source

Forgotten memories are soft-deleted first — confidence drops to zero and they never appear in recall. They stay on disk until physically purged, which gives you a chance to undo and lets the graph stay fast.

AI client routing

How aggressively Claude Desktop (and other MCP-aware AI clients) should reach for Graphnosis tools. With this on, the AI gets a system-prompt-level hint to use Graphnosis as your default memory layer — proactively calling recall before answering personal-context questions and correct instead of remember when fixing earlier statements.

File-source auto-reingest

When a file you've ingested changes on disk, Graphnosis can re-read it automatically — same flow as the Reingest button in the Sources tab. Useful for Obsidian notes, daily journals, or any file you actively edit. Off by default: enabling means every save fires a re-chunk + re-embed pass, which is fine for small notes but heavier for big PDFs.

Ingest performance

How aggressively the SDK splits documents into memory nodes, and how many texts it groups into each embedding model call. Sensible defaults cover most users; tune if ingest feels slow or recall feels imprecise. Changes apply to the next file you ingest — existing nodes keep their previous shape.

Quarantined files

Don't delete a quarantined file until you've successfully re-ingested its sources via Recover from op-log — it's the only on-disk copy of those bytes.
