                          Duplicate and redundant memories are merged automatically — no prompts. Pairs where a number, negation, or partial overlap made auto-merge unsafe are flagged for your call below. Nothing is ever lost: every merge is a soft-delete you can undo from the Recovery panel.

      🧠 Graphnosis Neural Network (.GNN)
      
                        
      🔮 A little neural net learns the shape of your graph and guesses connections you might've missed — hunches, not facts.
      
                        
      
                          
      🔒 .gai your real memories untouched
      
                          
      🧠 .gnn predicted edges — guesses lives here
      
                        
      
                        
        
                          
      • 🔮Predictions only — never your notes, never the canonical graph.
        • 
                          
      • 🔐Its own encrypted file beside your cortex: neural-network.gnn.
        • 
                          
      • ⏸️Off by default — a full snapshot is saved before it's ever enabled.
        • 
                          
      • 🗑️Delete the file → back to the exact pre-GNN state, byte-for-byte.
        • 
                        
      
                        
      
                      
      

                      
      
                        
      ✨ Graphnosis Local Layer (.GLL)
      
                        
      💡 Your on-device LLM quietly proposes links between related memories. You decide what sticks.
      
                        
      
                          
      🔒 .gai your real memories untouched
      
                          
      💡 .gll LLM's proposed links review below
      
                        
      
                        
        
                          
      • ✅Accept → it becomes a real edge.  ❌ Reject → gone for good.
        • 
                          
      • 🔐Encrypted overlay (local-layer.gll) — the LLM can't touch your attested memory.
        • 
                          
      • 🤖Needs Predict edges autonomously under Local LLM.
        • 
                          
      • 🗑️Delete the file → back to the pre-LLM state.
        • 
                        
      
                        
                        
      
                          
      .GLL autonomous edge prediction is a Pro feature
      
                          
      The background loop that asks your local LLM to propose
                          new connections between memories runs only with a Graphnosis Pro
                          subscription. The .gll overlay stays encrypted on your
                          device either way; Pro is what unlocks the engine that fills it.

                              Graphnosis can't find Ollama yet. Ollama is a free, open-source app
                              that runs the AI model on your own computer. It takes about three steps:
                            
      
                            
        
                              
      1. 
                                Install Ollama.
                                Download it from ollama.com →
                                Open the downloaded file and follow the installer — on a Mac, drag
                                Ollama into your Applications folder; on Windows, run the installer.
                              
        2. 
                              
      2. 
                                Start Ollama. Open the Ollama app once. It then runs
                                quietly in the background — look for its icon in the menu bar (Mac) or
                                system tray (Windows). There's no window to keep open; that's normal.
                                Ollama just needs to be running for Graphnosis to use it.
                              
        3. 
                              
      3. 
                                Connect. Come back here and click
                                Recheck above. When the status turns green, you can
                                download a model — Graphnosis walks you through it.
                              
        4. 
                            
      
                            
      
                              Ollama is free and open source, and runs entirely on your machine — your
                              memories never leave your device. Graphnosis only downloads the model
                              through your own copy of Ollama; it never installs anything for you.
                            
      
                          
      
                        
      

                        
                        
      
                          
      Prefer the Terminal?
      
                          
      
                            Pick your platform — Graphnosis auto-detects Ollama once it's
                            installed and running.
                          
      
                          
      
                            
                            
                            
                            
                            
                          
      

                          
      

      # Install via Homebrew (recommended)
brew install ollama

# Or download the .dmg from ollama.com and drag to Applications

# Start the background service
ollama serve   # (skip this if the menu-bar app is already running)
      
                          
      

                          
      

      # Option A — winget (Windows 10+ / PowerShell)
winget install Ollama.Ollama

# Option B — download the installer from ollama.com
# and run OllamaSetup.exe

# Ollama runs automatically as a Windows service after install.
# Confirm it's running:
ollama list
      
                          
      

                          
      

      # Official install script — works on most distros
curl -fsSL https://ollama.com/install.sh | sh

# The script installs Ollama as a systemd service and starts it.
# Confirm:
systemctl status ollama
ollama list
      
                          
      

                          
      

      # Ubuntu / Debian — same install script as Linux
curl -fsSL https://ollama.com/install.sh | sh

# If you prefer to skip the script and install manually,
# the script's contents are at https://ollama.com/install.sh

# Confirm the systemd unit is running:
sudo systemctl status ollama
ollama list
      
                          
      

                          
      

      # CPU-only

docker run -d --gpus=all -v ollama:/root/.ollama \
  -p 11434:11434 --name ollama ollama/ollama

# Pull / run inside the container
docker exec -it ollama ollama pull llama3.2:3b

      Search model
      
          
      
            Which on-device embedding model powers semantic recall. The default is
            tuned for English notes and is fast and lightweight. Switch to the
            multilingual model if you regularly store memories in more than one
            language — it dramatically improves cross-language recall but takes
            longer to download and uses more memory.

            A snapshot is taken first so the old vectors are recoverable.
          
      
          
      
            
          
      
          
      

          
      
          
      Ingest
      
          
      
            How aggressively the SDK splits documents into memory nodes, and how
            many texts it groups into each embedding model call. Sensible defaults
            cover most users; tune if ingest feels slow or recall feels imprecise.
            Changes apply to the next file you ingest. To apply the
            current chunk size to memory you've already saved, use
            Reingest all sources below.
          
      
          
      
            
            
          
      
          
      
            
            
          
      
          
      
            
            
            2 workers
          
      
          
      
            Parallel workers for the local embedding engine. 1 = lowest RAM / serial,
            2 = default, 3–4 = faster bulk ingest on capable machines. Change takes
            effect immediately without restarting.
          
      

          
      Reingest all sources
      
          
      
            Re-chunk and re-embed every source you've previously saved, using the
            current Chunk size, embedding model, and SDK improvements. Useful
            after changing chunk size or upgrading the search model. A snapshot
            is taken first so the old chunks/vectors are recoverable.
            

      
            Sources whose original content wasn't cached (e.g. you have
            Cache content set to off) are skipped — they'd need to be
            re-ingested from the source file or URL.
